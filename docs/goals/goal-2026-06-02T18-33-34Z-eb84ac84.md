Objective: Eliminate test churn caused by hardcoded live provider model ids in the valkyriweb/pi-mono fork, so a provider    
 dropping/renaming a model no longer breaks the test suite. Spike + rationale at /tmp/fork-bug2-live-model-pin-spike.md —     
 read it first.                                                                                                               
                                                                                                                              
 Outcome (what must be true when done):                                                                                       
 - packages/ai/test/** (and any other fork tests) no longer hardcode literal provider model ids (e.g.                         
   getModel("openai-codex", "gpt-5.5")) in a way that breaks when packages/ai/src/models.generated.ts changes.                
 - Behavior tests assert on a model capability derived from the registry (e.g. first model that supports images / xhigh) via  
   a small shared test helper; pure formatting/shape tests use a stable faux/fixture model. (Hybrid approach from the spike — 
   confirm or revise it before implementing.)                                                                                 
 - No production/core behavior change; only test code and test helpers. models.generated.ts is never hand-edited (change      
   packages/ai/scripts/generate-models.ts if generation must change).                                                         
                                                                                                                              
 Constraints:                                                                                                                 
 - Fork-owned files only; no upstream-native file edits; no core behavior delta. Customize via test helpers, not core.        
 - No any, erasable-TS-only syntax, no inline/dynamic imports, no single-call-site helpers.                                   
 - Commit only your own files with explicit paths; never git add -A. Don't push/merge/PR without explicit go.                 
                                                                                                                              
 Verification evidence (required before complete):                                                                            
 - npm run check passes with full output (no tail).                                                                           
 - Every touched test file run individually and passing (node ../../node_modules/vitest/dist/cli.js --run test/<file>).       
 - A package CHANGELOG or FORK-CHANGELOG.md entry where the guard requires it (test-only changes are exempt; helper/source    
   changes are not — verify with scripts/check-changelog-updated.mjs).                                                        
 - Demonstrate the churn is gone: simulate a dropped model and show the affected test still resolves a valid model instead of 
   failing.                                                                                                                   
                                                                                                                              
 Iteration policy: After each run, inspect failures, fix the root cause, re-run. Don't stop at a plan. If a long gate runs,   
 use monitor_start.

 ambiguous, ask once with concrete options, then proceed.                                                                     
                                                                                                                              
 Completion audit: Map every requirement above to fresh evidence (diffs, test output, check output) before marking complete.
