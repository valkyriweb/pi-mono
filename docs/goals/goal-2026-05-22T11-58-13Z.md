   ## What                                                                                               
                                                                                                         
   Extract the fork-only zoom-into-running-agent feature out of                                          
 `pi-mono-fork/packages/coding-agent/src/modes/interactive/interactive-mode.ts` into a new `pi-agent-ui` 
 workspace package under `~/Projects/personal/my-pi/packages/pi-agent-ui/`. The extension registers its  
 mount points via the B5 hooks (`registerMainPane`, `registerOverlay`, `registerFooter`) just landed in  
 `pi-mono-fork` commit `46fd9f3a`, and the zoom-related code is **removed from core** so core ships      
 smaller and the consumer proves the hook surface end-to-end.                                            
                                                                                                         
   Track: source code (zoom is genuinely fork-only — confirmed via git archaeology; commits `680577ec`   
 and `b26b2616` introduced it; not in upstream).                                                         
                                                                                                         
   ## Acceptance criteria (from issue 12)                                                                
                                                                                                         
   1. New workspace package `packages/pi-agent-ui/` with `package.json` declaring `pi.extensions =       
 ["./src/index.ts"]`.                                                                                    
   2. `ZoomedTaskComponent`, `ZoomedBashComponent`, `ZoomedSessionTranscript`, `getRunningTasksSorted`,  
 `cycleRunningTask`, `enterZoom`, `enterBashZoom`, `disposeZoomedComponent`, `preZoomChatChildren` and   
 the `app.agents.zoom.*` + `app.agents.footer.*` keybinding wiring all move into the extension. Core     
 retains only the generic B5 hook surface — no zoom-specific imports, types, fields, or methods anywhere 
 under `pi-mono-fork/packages/coding-agent/src/`.                                                        
   3. The extension registers the zoom main pane (`pi.registerMainPane('zoom', factory)`), the bash zoom 
 pane (`pi.registerMainPane('bash-zoom', factory)`), and the footer pill                                 
 (`pi.registerFooter('zoom-pill', spec)`). It uses `pi.showMainPane(id, payload)` /                      
 `pi.hideMainPane(id)` to drive activation.                                                              
   4. `interactive-mode.ts` calls `runner.bindSlotUI({ showMainPane, hideMainPane, showOverlay,          
 hideOverlay })` once at startup so the extension's imperative calls flow back into the chat container   
 component stack.                                                                                        
   5. Wired into the MacBook full profile via `packages/my-pi-full/package.json` `pi.extensions[]`.      
   6. **Unit tests:** zoom component lifecycle and the `getRunningTasksSorted` helper test moved with    
 the code; mock the B5 hook surface where needed.                                                        
   7. **Tmux smoke** verifies the full zoom round-trip still works end-to-end against the live `pi`      
 binary loading `pi-agent-ui`: `alt+z` enters zoom, `alt+l`/`alt+h` cycle, `alt+i` requestShutdown,      
 `alt+k` kill, `enter` injects, terminal-status auto-pop after `STOPPED_DISPLAY_MS`. Footer-nav          
 (`up`/`down`/`enter`/`escape` on empty editor) still works.                                             
   8. Headless mode (`pi -p`, `pi --mode rpc`) still works unchanged — extension registers nothing       
 useful but doesn't crash because B5 show/hide fall through to no-op stubs when `bindSlotUI` was never   
 called.                                                                                                 
   9. `npm run check` green in `pi-mono-fork`. `npm run check:profiles && npm run                        
 check:runtime-inventory` green in `my-pi`.
