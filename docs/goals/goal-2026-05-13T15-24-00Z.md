# Goal: TUI zoom-into-running-agent view (pi-mono-fork)                                                     
                                                                                                               
   ## Context already shipped (3 layers, all in main)                                                          
                                                                                                               
   Native `agent` tool now has a unified Task abstraction over background child runs.                          
   Read these in full before touching anything:                                                                
                                                                                                               
   - packages/coding-agent/src/core/tasks/         — Task adapter + ring buffer                                
     - types.ts            Task, TaskSnapshot, TaskStatus, isTerminalTaskStatus                                
     - local-agent-task.ts adapter over AgentRecentRun                                                         
     - registry.ts         listTasks, getTaskSnapshot, findTaskAdapter, subscribeTasks                         
     - messages.ts         appendTaskMessage, getTaskMessages, subscribeTaskMessages                           
     - index.ts            barrel                                                                              
   - packages/coding-agent/src/core/agents/executor.ts — already sinks live events                             
     into the ring buffer via `appendTaskMessage(taskId, …)` inside driveChildSession.                         
     taskId = AgentRecentRun.id.                                                                               
   - packages/coding-agent/src/core/tools/agent.ts — `action: "inject"` LLM surface                            
     (interrupt + resume(message) in one tool call).                                                           
                                                                                                               
   Existing per-run lifecycle in packages/coding-agent/src/core/agents/status.ts:                              
   - interruptAgentRecentRun(runId)  → soft stop, resumable                                                    
   - cancelAgentRecentRun(runId)     → hard kill                                                               
   - resumeAgentRecentRun(runId, prompt?)                                                                      
   - listAgentRecentRuns(), subscribeAgentRecentRuns(listener)                                                 
   - formatAgentStatus(), formatAgentFooterStatus()                                                            
                                                                                                               
   LocalAgentTask exposes verbs you should call from the TUI:                                                  
   - LocalAgentTask.kill(taskId)            → cancel                                                           
   - LocalAgentTask.requestShutdown(taskId) → interrupt (cooperative)                                          
   - LocalAgentTask.injectMessage(taskId, msg) → interrupt + resume(msg)                                       
                                                                                                               
   ## What to build                                                                                            
                                                                                                               
   A TUI "zoom" mode for any running background `local_agent` task:                                            
                                                                                                               
   1. **Footer pill** — show count of running tasks; add a hotkey to cycle through them.                       
      Existing footer plumbing: packages/coding-agent/src/core/footer-data-provider.ts                         
      plus formatAgentFooterStatus(). Read footer-data-provider.test.ts to understand                          
      the contract.                                                                                            
                                                                                                               
   2. **Zoom screen** — when zoomed into a task, swap the main transcript pane to                              
      render that task's live messages. Source: `subscribeTaskMessages(taskId, fn)`                            
      for live events, `getTaskMessages(taskId)` for the initial tail (≤200 events).                           
      Event kinds to render: assistant_text, assistant_end, tool_start, tool_end,                              
      user_injected. Existing transcript rendering lives in                                                    
      packages/coding-agent/src/modes/interactive/ — find the message-list                                     
      component and add a zoomed-task variant.                                                                 
                                                                                                               
   3. **Prompt input rebind while zoomed** — Enter calls                                                       
      `LocalAgentTask.injectMessage(taskId, text)` instead of parent submit.                                   
      After successful inject, append a `{kind:"user_injected", ts, text}` event                               
      into the task buffer (via appendTaskMessage) so it renders immediately.                                  
                                                                                                               
   4. **Keys (cooperative + hard)** while zoomed:                                                              
      - exit zoom → parent transcript                                                                          
      - cycle next/prev task                                                                                   
      - request shutdown (soft) → LocalAgentTask.requestShutdown                                               
      - kill (hard)             → LocalAgentTask.kill                                                          
                                                                                                               
      **NEVER hardcode key matches.** Add entries to DEFAULT_APP_KEYBINDINGS in                                
      packages/coding-agent/src/core/keybindings.ts. Read AGENTS.md rule before                                
      coding.                                                                                                  
                                                                                                               
   5. **Auto-pop on terminal** — when the zoomed task reaches                                                  
      completed|failed|cancelled|killed, hold the view for ~3 s (see Claude's                                  
      STOPPED_DISPLAY_MS pattern) showing final state, then pop back to parent.                                
      Use subscribeAgentRecentRuns or subscribeTaskMessages plus isTerminalTaskStatus.
