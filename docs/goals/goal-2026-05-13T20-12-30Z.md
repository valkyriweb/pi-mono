# Goal: TUI zoom view — footer navigation + real-session rendering

Builds on `goal-2026-05-13T15-24-00Z.md` (Phase 1, shipped). Phase 1 wired the zoom view, sort helper, keybindings, append-only render, and inject-without-await. Phase 2 makes the zoom feel like a real pi session you stepped into — not a transcript tail — and replaces the alt+z hotkey entry with arrow+Enter navigation that works in every terminal.

- `packages/coding-agent/src/core/tools/agent.ts` — `action:"inject"` surface
- `packages/coding-agent/src/core/agents/status.ts` — interrupt/cancel/resume + subscriptions; **`AgentRecentRun` only exposes status, not the live `AgentSession` object** — Phase 2b will need to extend this
- `packages/coding-agent/src/core/agents/executor.ts` — sinks `TaskMessageEvent`s via `appendTaskMessage(taskId, …)`; also the place where the child `AgentSession` is constructed
- `packages/coding-agent/src/modes/interactive/components/zoomed-task.ts` — Phase 1 component (append-only Text-per-event; perf fixed)
- `packages/coding-agent/src/modes/interactive/interactive-mode.ts` — `enterZoom`, `exitZoom`, `cycleZoom`, `injectIntoZoomedTask`, `scheduleZoomTerminalCheck`
- Keybindings: `app.agents.zoom.{enter,exit,cycleNext,cyclePrev,requestShutdown,kill}` defaulting to `alt+z / escape / alt+l / alt+h / alt+i / alt+k`
- Tests: `tasks-sort.test.ts` (helper) + `interactive-mode-zoom.test.ts` (component + integration, 8 cases)

## Build

### Phase 2a — Footer-navigable entry (smaller, ship first)

1. **Selectable footer pill**
   - `core/footer-data-provider.ts` already calls `formatAgentFooterStatus()`. Extend to render a *list-of-pills* layout when ≥1 running task — one pill per running task, sorted by `getRunningTasksSorted()`.
   - Highlight one pill at a time when "footer focus" is active. Highlighted pill uses an inverse/accent style.
2. **Footer focus state** in `InteractiveMode`
   - `footerSelectedTaskId: string | undefined` — `undefined` = not in footer-nav mode
   - Entered when user presses `up` while editor is empty *and* `getRunningTasksSorted().length > 0`
   - Exited on `escape`, on `enter` (which zooms), or when the editor regains text
3. **Keybindings** (add to `DEFAULT_APP_KEYBINDINGS`)
   - `app.agents.footer.focusPrev` — default `up` (only acts when editor empty + running tasks)
   - `app.agents.footer.focusNext` — default `down`
   - `app.agents.footer.zoom` — default `enter` (only acts when `footerSelectedTaskId` is set; otherwise fall through to normal submit)
4. **Wire to existing `enterZoom(taskId)`** — when `app.agents.footer.zoom` fires, call `enterZoom(footerSelectedTaskId)` and clear `footerSelectedTaskId`.
5. **No-conflict guarantees**
   - `up`/`down` must fall through to the editor's history navigation when editor is non-empty or no running tasks
   - `enter` must fall through to normal submit when `footerSelectedTaskId` is `undefined`
   - All conflict resolution lives in `custom-editor.ts` `handleInput` — same pattern as the existing `app.exit` ctrl+d gate
6. **Keep `alt+z` as power-user shortcut** (zooms most recent agent, no selection step)

### Phase 2b — Real-session render (replace Text-per-event tail with proper message components)

Goal: when you zoom, you see the child agent's transcript rendered identically to the parent session — streaming assistant text mutates one message in place (no duplicate-prefix lines), tool calls show full `ToolExecutionComponent` widgets with expandable output, edit diffs, the works.

1. **Expose the child `AgentSession` on `AgentRecentRun`**
   - Today `AgentRecentRun` only carries status + lifecycle. Phase 2b adds an optional `session: AgentSession | undefined` field set by `executor.ts` when the run starts, cleared on terminate.
   - `findAgentRecentRun(taskId)?.session` becomes the entry point for live message access.
2. **Build `ZoomedSessionTranscript` component** in `src/modes/interactive/components/`
   - Reuses the existing message rendering pipeline. Look at how `InteractiveMode` builds `AssistantMessageComponent` / `UserMessageComponent` / `ToolExecutionComponent` from `runnerCtx.transcript` and mirror it.
   - Subscribes to the child session's event stream (`session.subscribe(event => …)`) instead of `subscribeTaskMessages`. Same events (`message_update`, `tool_execution_start`, `tool_execution_end`) — but now we mutate in-place because we hold the same `AssistantMessageComponent` instance across text deltas.
   - Falls back gracefully if `findAgentRecentRun(taskId).session` is undefined (terminal task, replay-only) — show the `TaskMessageEvent` tail from Phase 1 as a static history.
3. **Replace `ZoomedTaskComponent` body** with `ZoomedSessionTranscript` when a live session is available; keep the existing component as the terminal-state fallback.
4. **Inject still goes through `LocalAgentTask.injectMessage`** — no executor changes.
5. **Auto-pop on terminal** stays in `InteractiveMode` (already wired).
